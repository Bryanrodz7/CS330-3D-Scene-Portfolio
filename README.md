# CS 330 - Computational Graphics and Visualization Portfolio

This repo has my final project for CS 330. It's a 3D scene built in OpenGL/C++, plus the Design Decisions document that explains how I built it.

## Contents

- **3D Scene ZIP** - source code and project files for the 3D scene
- **Design Decisions Document** - explains why I made the choices I made for the objects, layout, and textures/lighting

## Reflection

### How do I approach designing software?

For this project I chose a kids park, since it's somewhere I go a lot with my kids and they liked watching me build it. I looked up the actual park's layout and tried to match it as close as I could instead of just making something up. That changed how I designed the whole scene. I wasn't just placing shapes wherever, I was trying to get the real positions and proportions right, the floor, the walls, the playground equipment, all of it. That made me actually plan things out first instead of just jumping in and building. Going forward I'll use that same approach on other projects too, plan it out first, then build piece by piece and check it against what you're going for.

### How do I approach developing programs?

I built this one piece at a time and tested after every change instead of writing a bunch of code and hoping it worked. Add a shape, run it, see if it looks right, fix it if it doesn't, move to the next thing. Lighting and textures especially took a few tries to get looking right, nothing worked the first time. By the end of the milestones I got a lot more comfortable just testing right after every change instead of waiting until the end to see if anything broke.

### How can computer science help me in reaching my goals?

This gave me real hands on experience with how 3D graphics actually work, positioning objects, lighting, textures, getting it all to render right together. I'm still not 100% set on which direction I want to go in CS, but having this kind of graphics experience gives me more to work with either way, whether that ends up being regular software work, something more game or graphics related, or just knowing more about how this stuff gets built. The habit I picked up here, build small and test as you go, is something I'll keep using no matter what I end up doing.
